# Mosaic 核心概念指南

> **文档用途**: 为用户提供 Mosaic 项目的高层概念概览，帮助快速理解系统功能与价值。
> **目标读者**: 最终用户、系统管理员、Agent 开发者

---

## 1. 什么是 Mosaic?

**Mosaic** 是一个**分布式的多智能体协作网格 (Mesh)**。

想象一个由多个 AI 智能体（Agents）和系统组件组成的“团队”。在 Mosaic 中，这些成员不再是孤立的聊天机器人，而是连接在一个共同的通信网络中。它们可以互相发送消息、监控彼此的行为、协同完成复杂的任务，就像将一块块独立的马赛克拼贴成一幅完整的图画。

### 核心价值
*   **打破孤岛**: 让 Claude、Gemini、Cursor 等不同来源的智能体能够互相协作。
*   **透明监督**: 任何智能体的行为（如工具调用）都可以被其他智能体（如审计员）实时监控和干预。
*   **灵活编排**: 通过简单的“订阅”机制，即可定义复杂的协作流程，无需硬编码。

---

## 2. 核心模型：网格与节点

Mosaic 的世界观由两个基础概念构成：**网格 (Mesh)** 和 **节点 (Node)**。

### 2.1 网格 (Mesh)
网格是协作的边界。它就像一个“虚拟办公室”或“局域网”。
*   同一个网格内的成员可以互相“看见”并通信。
*   不同网格之间是完全隔离的。
*   所有的协作关系（拓扑）都保存在网格中。

### 2.2 节点 (Node)
节点是网格中的成员，也是你主要交互的对象。每个节点代表一个独立的实体。

**节点的类型：**
*   **Agent 节点**: 拥有大模型能力的智能体（如 `worker`, `auditor`）。它们能理解自然语言，执行推理，使用工具。
*   **组件节点**: 传统的功能组件（如 `scheduler` 定时器, `webhook` 接收器）。它们按规则办事，负责连接外部世界。

**关键特性：**
*   **独立性**: 每个节点都是独立的，拥有自己的身份 ID。
*   **角色定义**: 节点的具体职责（如“负责写代码”或“负责审查代码”）由你通过 **Program 模式** 赋予。

---

## 3. 协作机制：事件与订阅

节点之间如何协作？它们不打电话（API 调用），而是广播**事件 (Event)**。

### 3.1 事件 (Event)
事件是节点发出的“信号”。
*   **Hook 事件**: 当智能体想做动作时触发。例如，`PreToolUse`（我准备使用工具了）、`UserPromptSubmit`（用户跟我说话了）。
*   **消息事件**: 节点主动发送的消息。

### 3.2 订阅 (Subscription)
订阅定义了“谁关心什么”。这是 Mosaic 最强大的编排机制。

*   **普通订阅 ("告诉我一声")**: 
    *   *例子*: Logger 节点订阅了 Worker 的所有事件。
    *   *效果*: Worker 做任何事，Logger 都会收到通知并记录下来。Worker 不需要等待。
    
*   **阻塞订阅 ("必须经过我同意")**: 
    *   *例子*: Auditor（审计员）订阅了 Worker 的 `!PreToolUse` 事件。
    *   *效果*: 当 Worker 试图使用工具（如删除文件）时，必须**暂停**，等待 Auditor 批准。如果 Auditor 拒绝，Worker 的操作就会失败。
    *   *价值*: 这是实现**安全监管**和**Human-in-the-loop**的基础。

---

## 4. 智能体的工作流：会话 (Session)

在 Mosaic 中，智能体不是无状态的函数，而是基于**会话**工作的。

### 4.1 什么是会话？
会话是智能体的一段连续记忆流。它包含了对话历史、上下文和当前的任务状态。

### 4.2 灵活的会话路由策略
Mosaic 提供了多种策略来决定事件应该由哪个会话处理：
*   **镜像模式 (Mirroring)**: 最常用的“跟随”策略。针对上游的每一个会话，我也开启一个对应的会话。就像是上游智能体的“影子”或“副驾驶”。
*   **聚合模式 (Aggregation)**: “多源汇聚”。忽略上游的来源，将所有事件都汇总到同一个长期会话中处理（例如日志汇总、运维中心）。
*   **任务模式 (Tasking)**: “一事一议”。每个事件都是一个独立的任务，处理完即销毁，不保留长期记忆。

## 5. 节点的三种状态

为了适应不同的使用场景，节点可以在三种状态下切换：

### 5.1 交互模式 (Interactive Mode)
*   **场景**: 你想亲自与某个智能体聊天，或者看着它干活。
*   **行为**: 启动一个终端界面，你可以直接输入指令。同时，它依然连接在 Mesh 中，受其他节点监督。
*   *命令*: `mosaic run <node_id>`

### 5.2 后台模式 (Background Mode)
*   **场景**: 智能体作为“后台服务”默默工作（如日志记录、定时任务、审计）。
*   **行为**: 没有界面，在后台自动运行。当收到订阅的事件时自动唤醒处理。
*   *命令*: `mosaic start <node_id>` (通常由系统自动管理)

### 5.3 编程模式 (Program Mode)
*   **场景**: 你需要“培训”一个新的智能体上岗。
*   **行为**: **断开**与 Mesh 的连接（防止误操作）。你通过对话告诉节点：“你是审计员，你的职责是检查删除命令...”。配置完成后，这些知识会被保存下来。
*   *命令*: `mosaic program <node_id>`

---

## 6. 总结

Mosaic 让你能够：
1.  **创建** 各种角色的智能体节点。
2.  通过 **订阅** 将它们连接起来，定义数据流向和控制权。
3.  通过 **编程模式** 赋予它们具体的职责。
4.  最终形成一个**自治、可控、可协作**的智能体网络。
